## Title

CARE Machine: Source-Grounded Recursive Canon Architecture

## Purpose

This document records the concept, architecture, and intended use of the CARE Machine / CARE Library in order to establish prior art and prevent enclosure of the basic method by later proprietary claims.

The aim is not to prevent use, study, copying, forking, or extension.

The aim is to prevent a later closed system from enclosing the concept of a source-grounded recursive canon machine and presenting it as proprietary.

## Summary

CARE — the Canonical Axiomatic Reduction Engine — is a source-grounded recursive architecture for processing books and other texts into layered conceptual structures.

The system creates structured units at multiple scales:

- RAW source units;
- RAW passage units;
- chapter-level CARE units;
- cluster-level CARE units;
- metacluster-level CARE units;
- whole-book CARE units;
- corpus-level CARE units;
- cross-lineage synthesis units;
- strange-child synthesis units;
- optional overlay units.

## Core method

The basic method is:

1. ingest a source text;
2. clean and normalise the text;
3. create a RAW unit;
4. split the RAW unit into source passages;
5. assign stable passage identifiers;
6. generate chapter-level interpretive units;
7. generate cluster-level syntheses from chapter units;
8. generate metacluster syntheses from cluster units;
9. generate whole-book syntheses from metacluster or cluster units;
10. preserve ancestry and source references across levels;
11. allow multiple books to be compared through corpus or cross-lineage synthesis.

## Passage provenance

CARE units should preserve or inherit source ancestry.

A high-level claim may be traced through:

```text
WholeBook unit
→ Metacluster unit
→ Cluster unit
→ Chapter unit
→ RAW passage
→ source text
```

This makes CARE different from ordinary summarisation. The system does not merely compress content. It creates a layered conceptual architecture with receipts.

## Forkable canon model

The CARE Library is intended as a forkable canon.

This means that different users, researchers, communities, or institutions may generate their own CARE-compatible libraries and compare them.

A CARE-compatible library should preserve:

- unit identifiers;
- source ancestry;
- processing stage;
- version information;
- passage provenance where possible;
- correction and fork history.

## Anti-enclosure purpose

The purpose of this defensive publication is to record the basic architecture of CARE as prior art.

This includes:

- recursive movement from RAW to CH to CL to MC to WB;
- passage-grounded provenance;
- corpus-level comparison;
- cross-lineage synthesis;
- forkable canon infrastructure;
- public or semi-public CARE-compatible libraries;
- open correction and extension of interpretive units.

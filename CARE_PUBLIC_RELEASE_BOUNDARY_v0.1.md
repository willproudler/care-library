# CARE Library Public Release Boundary v0.1

**Status:** Public release-control policy  
**Scope:** CARE Library v0.1 documentation

## Current public scope

This repository publishes only approved CARE-authored documentation describing the CARE v0.1 architecture and its defensive-publication claims.

No CARE corpus, source-book collection, RAW text collection, or private CARE Library dataset is released by this repository.

## Excluded material

The following material must not be committed, tagged, attached to a release, or deposited publicly unless it completes a separate rights and release review:

- private source books and source documents;
- RAW texts, passages, annotations, and corpus databases;
- private corpus inventories and work-level processing records;
- generated outputs derived from sources that are not cleared for public release;
- unpublished specifications beyond the approved v0.1 disclosure;
- trademark filing evidence, receipts, correspondence, and payment records;
- API keys, credentials, local configuration, personal information, and private filesystem paths;
- third-party text, images, data, or other material without a documented public-release basis.

## Release rule

A file may enter a public release only when:

1. it is explicitly included in the release manifest;
2. its rights basis has been reviewed;
3. it contains no secrets, private paths, or unnecessary personal information;
4. its final checksum is recorded; and
5. it has been reviewed from a clean copy of the proposed release.

Presence in the repository does not by itself approve a file for release. `.gitignore` reduces accidental additions but is not a security boundary.

## Licence boundary

CARE licences apply only to material expressly identified as CARE-authored or otherwise licensed for release. They do not grant rights in third-party source texts or private CARE Library contents.

## Repository history

Earlier public history may contain withdrawn drafts or inventory metadata. Removal from the current branch does not erase that history or change the rights status of earlier material.

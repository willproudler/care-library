# CARE Commons Charter v0.1

## Author

William Proudler

## Principle

CARE is intended as a commons-oriented canon machine.

The purpose is not to prevent use, copying, forking, study, or extension.

The purpose is to prevent enclosure, misrepresentation, and capture of the CARE Library as a closed proprietary interpretive product.

## CARE is not

CARE is not:

- a proprietary summary engine;
- a closed syllabus;
- an institutional canon;
- a final authority over texts;
- a platform-owned reading product;
- a replacement for books;
- a replacement for scholarship.

## CARE is

CARE is:

- a source-grounded recursive library machine;
- a way of making books structurally comparable;
- a forkable canon architecture;
- a traceable interpretive process;
- a commons protocol for computational hermeneutics.

## CARE-compatible systems

A CARE-compatible system should preserve:

- source ancestry;
- visible processing stages;
- passage provenance where possible;
- correction history;
- forkability;
- distinction between source text and generated interpretation;
- version information;
- uncertainty and failure markers.

## Forkability

CARE should be forkable.

This means that different users, scholars, artists, institutions, or communities may produce different CARE-compatible libraries.

Disagreement should not be erased.

Multiple CARE libraries should be able to exist and be compared.

## Anti-enclosure

The CARE name and originating architecture should be protected against capture.

A closed system that does not preserve traceability, forkability, or source ancestry should not represent itself as the CARE Library.

## Open canon

CARE revives the canon as an open-source structure.

The canon should not be a final list of authorised books.

It should become a revisable, inspectable, source-grounded field of relations.

## Long-term aim

The long-term aim is an open canon infrastructure: a way for books, concepts, traditions, contradictions, and excluded lineages to be mapped without being owned by a single institution, platform, or authority.

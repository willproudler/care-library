# Security Policy

## Scope

CARE Library v0.1 currently publishes documentation and defensive-publication material rather than operational software.

Security reports may nevertheless concern accidental disclosure, credentials, personal information, private source material, repository configuration, or release-integrity problems.

## Private reporting

Do not open a public issue if a report might reveal:

- credentials or secret configuration;
- private CARE Library material;
- source books, RAW texts, or corpus data;
- personal or transaction information;
- unpublished specifications;
- instructions that would increase the impact of an exposure.

Use GitHub's private vulnerability-reporting feature through the repository's **Security** tab.

If private reporting is unavailable, open a public issue requesting a private contact route, but include no sensitive details or attachments.

## Report contents

Where safe, identify:

- the affected public file, URL, release, or commit;
- the type and likely impact of the problem;
- whether the material is still publicly accessible;
- a minimal reproduction that contains no private source material.

Do not upload private evidence, credentials, copyrighted source texts, or personal information.

## Handling

Reports will be assessed before public discussion. Remediation may include removing material from the current branch, revoking credentials, correcting release metadata, or preparing a new release version.

Removing a file from the current branch does not erase earlier Git history.
